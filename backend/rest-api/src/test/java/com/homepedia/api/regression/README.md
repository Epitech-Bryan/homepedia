# Regression Tests — DO NOT MODIFY WITH AI AGENTS

These tests protect critical business logic. They serve as contract tests
that verify exact expected outputs for known inputs.

If a regression test fails after a code change:
1. The code change likely broke business logic — investigate and fix the code
2. Do NOT update the test to match new behavior without human review
3. Only a human should modify these tests after verifying the business logic change is intentional
